PROJECT 01 · PYTHON 100% · MIT LICENSE

Macro Quant Dashboard

A full-stack quantitative finance dashboard built with Streamlit. It pulls live equity and macro data from Yahoo Finance, things like Brent crude, 10-year yields, the VIX and USD/NOK, then tests whether those factors actually predict next-day stock returns.

The interesting part is the modelling. I run multivariate OLS regressions with HC1 robust standard errors to check for statistically significant relationships, then layer a Random Forest on top to catch the non-linear patterns the regression misses and rank which macro factors matter most. There is also a spotlight feature that loops through the portfolio and flags individual stocks showing significant momentum effects.

This project sits closest to where I want to go next: turning messy market data into something a decision maker can act on. Useful background for both the shipping side of my degree and the finance master I am planning.

PROJECT 02 · PYTHON 100% · MIT LICENSE

Chess Trainer

A complete chess training application with Stockfish engine integration, built with Tkinter and python-chess. You can play against the engine at adjustable strength from 800 to 2500+ ELO, solve tactical puzzles pulled from Lichess datasets filtered by theme and rating, and review finished games with centipawn accuracy analysis.

The game review assigns move badges like Best, Mistake and Blunder, similar to what the big chess platforms do, and the app saves your ELO, streaks and game state between sessions. On first run it processes the raw Lichess puzzle CSV into a lightweight local database so puzzle loading stays fast.

This was the project where I learned to handle real complexity: engine communication, state persistence, dataset processing and a responsive UI all living together in one application.

PROJECT 03 · PYTHON 100%

Batch AVI to MP4 Converter

A desktop GUI tool built with Tkinter and FFmpeg that converts batches of heavy .avi files into widely compatible .mp4. Conversions run on background threads, two at a time, so the interface never freezes while files are processing, and each file gets its own real-time progress bar.

The small details are what make it useful. It enforces a standard pixel format so the output plays correctly in Windows Media Player without throwing codec errors, it never deletes your source files, and it handles missing assets gracefully instead of crashing. I also wrote installation instructions covering Windows, Mac and Linux, including the manual PATH setup for people whose package managers fail.

It started because I had a folder of old videos that nothing would play. Instead of paying for converter software, I built one.
